Frequently Asked Questions

General Questions

What technologies did you use to build this website?

This website is built using simple, standard web technologies: HTML5 for structure, CSS3 for styling, and JavaScript for interactivity. I also use Markdown for content management to keep things simple and maintainable.

Why did you choose not to use a framework?

I believe in the power of simplicity. While frameworks like React, Vue, or Next.js are excellent for complex applications, they can be overkill for simple websites. Using basic web technologies results in faster loading times, better performance, and easier maintenance.

Is this website responsive?

Yes! The website is fully responsive and works great on desktop, tablet, and mobile devices. The CSS uses modern techniques like CSS Grid and Flexbox to ensure a good experience across all screen sizes.

Content Management

How do you manage blog content?

I write blog posts in Markdown format, which is simple and readable. The JavaScript on the blog page converts the Markdown to HTML and displays it. This makes it easy to write and edit content without dealing with complex content management systems.

Can I add new blog posts easily?

Absolutely! To add a new blog post, simply create a new Markdown file in the `src/blog/` directory. The blog system will automatically detect and display it. The format is very simple - just write your content in Markdown and add some metadata at the top.

Technical Questions

How do you handle SEO?

Since this is a static website, SEO is handled through proper HTML structure, meta tags, and semantic markup. The site loads quickly and is easily crawlable by search engines.

Do you use any external dependencies?

No external dependencies are required! Everything is built with vanilla HTML, CSS, and JavaScript. This makes the site fast, secure, and easy to deploy anywhere.

How can I deploy this website?

You can deploy this website to any static hosting service like GitHub Pages, Netlify, Vercel, or any web server. Since it's just HTML, CSS, and JavaScript files, it's very portable and easy to host.
